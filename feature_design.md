# 医疗档案管理APP — 产品需求文档（PRD）& 功能设计

> 版本：v2.0  
> 更新日期：2026-04-09  
> 定位：面向非技术用户的个人医疗档案管理工具，核心交互为语音+对话

---

## 一、产品概述

### 1.1 产品背景

用户频繁就医，但每次的病历、检查单、处方等记录分散难统一，后续查询十分不便。本产品旨在提供一个随时记录、智能检索的医疗档案管理工具，让用户用最自然的方式（语音/文字）管理自己的健康数据。

### 1.2 目标用户

| 维度 | 描述 |
|------|------|
| 核心用户 | 经常就医、需要统一管理病历的中青年及中老年人群 |
| 技术水平 | 无代码背景，对复杂操作有抵触心理 |
| 使用场景 | 医院候诊时快速记录、回家后补充详情、复诊前查询历史 |
| 核心诉求 | 简单易用、语音优先、一键记录、智能问答 |

### 1.3 产品目标

1. **零学习成本**：用户打开APP即可使用，无需理解页面结构
2. **语音优先**：长按即可语音输入，降低文字输入门槛
3. **智能关联**：自动从语音/文字中提取医疗信息，无需手动填表
4. **随时查询**：自然语言问答，快速检索历史记录

## 二、交互架构设计

### 2.1 整体交互原则

> **没有首页、没有Tab页。一个主操作区域，一切从简。**

| 原则 | 说明 |
|------|------|
| 单一入口 | 登录后直接进入主操作区域，无多余跳转 |
| 语音优先 | 页面中央突出语音按钮，长按即录 |
| 渐进展示 | 其他功能收在侧边抽屉，不干扰主流程 |
| 对话即操作 | 所有交互（记录、查询、导航）通过对话完成 |

### 2.2 页面结构

```
┌─────────────────────────────┐
│  ☰  医疗档案管理        ⚙️  │  ← 顶部栏（左：抽屉入口 / 右：设置）
├─────────────────────────────┤
│                             │
│  聊天消息区域               │  ← 深色背景，消息气泡
│  用户消息靠右（琥珀黄）     │
│  系统消息靠左（深色表面）   │
│                             │
│                     🎤      │  ← 悬浮语音按钮（80px，右下角固定）
│                  长按语音   │
├─────────────────────────────┤
│  🖼️  [输入您的问题...]  发送 │  ← 底部输入栏（图片上传 + 文字输入 + 发送）
└─────────────────────────────┘
```

### 2.3 侧边抽屉

点击左上角 ☰ 按钮，从左侧滑出抽屉面板：

```
┌──────────────┐
│ 功能菜单    ✕ │  ← 抽屉头部（渐变背景）
├──────────────┤
│ 📋 医疗记录   │  → 跳转记录列表页
│ 📊 健康统计   │  → 跳转统计页
│ ➕ 添加记录   │  → 跳转手动录入页
│ 📤 上传资料   │  → 触发文件选择
└──────────────┘
```

### 2.4 交互流程

#### 主流程：语音记录
```
长按语音按钮 → 录音开始（按钮变红+脉冲动画）
    → 说话（最长2分钟）
    → 松开按钮 → 语音识别文字回填至输入框
    → 用户确认后手动发送
    → 自动提取医疗信息（医院/科室/诊断）
    → 生成就诊记录 + 对话回复确认
```

#### 语音取消流程
```
长按语音按钮 → 录音中
    → 手指/鼠标上划超过80px → 进入取消状态（按钮变灰+提示"松手取消"）
    → 松手 → 取消本次识别，清空输入框
    → 若上划后下划回80px以内 → 恢复录音状态
```

#### 主流程：文字问答
```
输入框输入问题 → 发送
    → 系统理解意图
    → 检索相关记录
    → 对话区域显示回答
```

#### 主流程：图片上传
```
点击图片按钮 → 选择本地图片
    → 图片上传至聊天区域
    → （后续）OCR识别提取信息
```

#### 语音命令识别规则

| 语音关键词 | 触发动作 |
|-----------|---------|
| "添加"/"记录" | 解析医疗信息，创建就诊记录 |
| "删除"/"删掉"/"移除" | 识别删除意图，删除指定记录 |
| "查看"/"历史" | 跳转医疗记录列表页 |
| "统计" | 跳转健康统计页 |
| 其他内容 | 作为问题进行智能问答 |

## 三、功能模块详细设计

### 3.1 用户认证模块

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 手机号验证码登录 | 输入手机号，获取验证码后登录（模拟模式验证码：123456），新手机号自动注册 | P0 |
| 微信登录 | 点击微信图标一键登录（模拟模式），新用户自动注册 | P0 |
| 自动登录 | 记住登录状态（JWT令牌），下次打开直接进入主界面 | P0 |
| 退出登录 | 设置按钮点击后确认退出 | P1 |

**技术实现**：
- 后端 JWT 令牌认证，令牌有效期7天
- 手机号验证码登录：模拟模式固定验证码123456，后续接入真实短信服务
- 微信登录：模拟模式自动创建用户，后续接入微信开放平台
- 前端 localStorage 存储 JWT 令牌和用户信息
- 后端 API：/api/auth/send-code、/api/auth/login-phone、/api/auth/login-wechat、/api/auth/me

### 3.2 主操作区域（对话+语音）

这是APP的核心界面，融合了语音输入、文字输入、对话展示三大功能。

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 悬浮语音按钮 | 右下角80x80px悬浮按钮，长按开始录音 | P0 |
| 录音状态反馈 | 按钮变红+脉冲动画+录音计时显示 | P0 |
| 语音取消 | 录音中上划80px取消，松手清空识别内容 | P0 |
| 语音识别 | 使用Web Speech API（浏览器）/ Capacitor原生插件（App），支持中文 | P0 |
| 语音命令解析 | 识别"添加记录"/"查看历史"等关键词 | P0 |
| 文字输入 | 底部输入框，支持回车发送，Shift+Enter换行，兼容中文输入法 | P0 |
| 图片上传 | 点击图片按钮选择本地图片 | P1 |
| 对话消息展示 | 气泡式消息，区分用户/系统 | P0 |

**交互细节**：
- 语音按钮：悬浮于右下角（bottom:112px），长按300ms后开始录音，松开停止，最长2分钟（120秒）自动停止
- 录音计时：录音中语音按钮标签实时显示已录制秒数（如"5"、"1:03"），到达2分钟自动停止
- 松手行为：松开后识别文字回填至输入框，不自动发送，用户确认后手动点击发送
- 上划取消：录音中手指/鼠标向上滑动超过80px进入取消状态，按钮变灰+提示"松手取消"；松手后清空识别内容；下划回80px以内可恢复录音
- 连续识别：使用 continuous 模式，支持长时间持续语音输入，中间停顿不会中断录音
- 实时预览：录音过程中识别到的文字实时显示在输入框中
- 消息区域：用户消息靠右（琥珀黄），系统消息靠左（深色表面层级1）
- 语音按钮在非主界面时自动隐藏

### 3.3 医疗记录管理模块

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 语音添加记录 | 通过语音输入自动解析并创建记录 | P0 |
| 手动添加记录 | 抽屉→添加记录，填写表单 | P0 |
| 查看记录列表 | 按时间倒序展示所有记录 | P0 |
| 筛选记录 | 按时间范围/关键词筛选 | P1 |
| 查看记录详情 | 点击记录查看完整信息 | P1 |
| 编辑记录 | 修改已有记录内容 | P2 |
| 删除记录 | 通过对话或界面删除指定记录 | P0 |

**记录数据结构**：

```json
{
    "id": "时间戳",
    "date": "就诊日期",
    "hospital": "医院名称（AI提取，未识别则显示"未知医院"）",
    "department": "科室",
    "doctor": "医生",
    "complaint": "主诉/症状（AI总结，非用户原话）",
    "diagnosis": "诊断",
    "medication": "用药信息",
    "examination": "检查项目",
    "notes": "备注",
    "images": ["图片ID列表"]
}
```

**卡片展示规范**（会话内卡片和抽屉列表统一）：

| 展示元素 | 说明 |
|---------|------|
| 医院名称 | 标题行，AI提取，未识别显示"未知医院" |
| 日期 · 科室 | 副标题行，科室可选，无科室则只显示日期 |
| 主诉/症状 | 正文区域，AI总结的医学术语，非用户原话 |

**详情页展示规范**（结构化信息展示）：

| 字段 | 图标 | 展示规则 |
|------|------|---------|
| 医院名称 | — | 顶部居中标题 |
| 日期 · 科室 | — | 顶部居中副标题 |
| 主诉/症状 | 🩺 听诊器 | 有值时显示 |
| 诊断结果 | 📋 医疗文件 | 有值时显示 |
| 医生 | 👨‍⚕️ 医生 | 有值时显示 |
| 用药信息 | 💊 药片 | 有值时显示 |
| 检查项目 | 🧪 试管 | 有值时显示 |
| 备注 | 📌 便签 | 有值时显示 |
| 就诊图片 | — | 有图片时显示缩略图 |

**多轮对话记录合并规则**：

| 场景 | 处理方式 |
|------|---------|
| 同一次就诊的补充信息 | 合并到同一条记录（AI判断 is_new_visit=false） |
| 明确提到另一次就诊 | 创建新记录（AI判断 is_new_visit=true） |
| 不确定时 | 默认合并到当前记录 |

**AI提取规则**：
- 医院名称：完全由AI识别，不使用正则匹配，未识别则显示"未知医院"
- 主诉/症状：AI用简洁医学术语总结，不照搬用户原话
- 诊断结果：未明确则为空，不显示"待确认"
- 用药信息、检查项目：AI从对话中提取

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 自然语言问答 | 在主界面直接输入问题 | P0 |
| 语音提问 | 长按语音按钮说出问题 | P0 |

| 问题类型 | 示例 | 回答逻辑 |

| 诊断查询 | "我有什么诊断记录？" | 列出所有诊断结果 |

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 图片上传 | 从相册选择或拍照上传 | P1 |
| 图片预览 | 在对话中显示上传的图片 | P1 |

1. 点击输入框旁的图片按钮
2. 选择本地图片文件
3. 图片显示在对话区域
4. （后续）OCR识别并提取信息
5. 用户确认后关联到就诊记录

|--------|------|--------|

### 3.7 数据管理模块

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 本地数据存储 | localStorage + IndexedDB 持久化存储 | P0 |
| 云端数据存储 | 后端 SQLite 数据库，通过 API 同步 | P0 |
| 多端数据同步 | pull/push 双向同步，版本号冲突解决 | P1 |
| 离线缓存 | 离线操作本地保存，联网后自动推送 | P1 |
| 数据备份 | 导出备份文件 | P2 |
| 数据恢复 | 从备份文件恢复 | P2 |
| 数据清除 | 清除所有本地数据 | P2 |
| 审计日志 | 记录用户操作（登录/创建/修改/删除/同步） | P2 |

**同步机制**：
- 登录后自动启动30秒间隔同步
- 先推送本地变更（push），再拉取服务端更新（pull）
- 基于版本号的冲突解决：服务端版本优先
- 后端 API：/api/sync/pull、/api/sync/push

## 五、技术实现方案

### 5.1 技术栈（当前版本）

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | HTML5 + CSS3 + JavaScript | 纯Web，无框架依赖 |
| 后端 | Node.js + Express | AI代理+静态文件服务 |
| 语音识别 | Web Speech API + Capacitor原生插件 | 浏览器原生 + 原生语音识别 |
| AI服务 | DeepSeek API（后端代理） | API Key不暴露在前端 |
| 数据存储 | localStorage | 本地键值对存储（后续迭代上云） |
| 壳应用 | Capacitor 6+ | 打包为iOS/Android |

### 5.2 技术栈（后续迭代）

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | React Native / Flutter | 跨平台原生应用 |
| 语音识别 | 系统原生API + 云端ASR | 更高识别准确率 |
| 数据库 | SQLite | 结构化本地存储 |
| OCR | Tesseract / ML Kit | 图片文字识别 |
| AI问答 | 本地轻量模型 + 云端API | 智能问答能力 |
| 加密 | AES-256 | 敏感数据加密存储 |

### 5.3 关键技术要点

#### 语音处理
- 集成Web Speech API（浏览器）/ Capacitor原生语音识别插件（App）
- 支持中文语音识别，连续识别模式
- 最长2分钟录音，自动停止
- 语音命令关键词匹配：添加/记录/查看/历史/统计

#### 数据存储
- 原型阶段：localStorage JSON序列化
- 正式版：SQLite结构化存储 + 文件加密
- 数据按用户ID隔离，支持多用户

#### 图片处理
- 原型阶段：文件选择 + 预览
- 正式版：相机集成 + 图片预处理 + OCR识别

#### 安全隐私
- 本地数据加密存储
- 密码哈希处理
- 敏感信息脱敏显示
- 后续：生物识别认证
